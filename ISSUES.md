# ShortLink – GitHub Issues Backlog

The following table lists all planned GitHub Issues for the initial development phase of the **ShortLink** URL shortening application (Blazor frontend · C# backend · SQLite database).

| # | Title | Description | Technical Stack | Priority |
|---|-------|-------------|-----------------|----------|
| 1 | SQLite database setup and schema design | Initialize the SQLite database, configure the EF Core provider, and define the overall schema structure. Set up connection string management and ensure the database file is created on first run. | C# | High |
| 2 | Links table implementation | Create the `Links` entity and corresponding database table with fields: `Id`, `ShortCode`, `OriginalUrl`, `CustomAlias`, `CreatedAt`, `ExpiresAt`, and `IsActive`. Add indexes on `ShortCode` and `CustomAlias` for fast lookups. | C# | High |
| 3 | Click events table implementation | Create the `ClickEvents` entity and corresponding database table with fields: `Id`, `LinkId` (FK), `ClickedAt`, and any additional metadata fields (e.g. referrer). Ensure a cascade-delete relationship with the `Links` table. | C# | High |
| 4 | Database migrations setup | Configure EF Core migrations, create the initial migration covering the `Links` and `ClickEvents` tables, and apply it automatically on application startup (or via a dedicated migration command). | C# | High |
| 5 | URL shortening logic – random code generation and custom alias handling | Implement the service that generates a unique random short code (e.g. 6 alphanumeric characters) and also accepts an optional custom alias. Ensure uniqueness checks against the database before persisting a new link. | C# | High |
| 6 | Click tracking logic | Implement dedicated click-tracking logic that records a new `ClickEvent` row every time a short link is resolved. The service must be called atomically with the redirect response to guarantee no clicks are lost. Include total-click aggregation used by the analytics endpoints. | C# | High |
| 7 | Validation service for URLs and aliases | Create a validation service that checks: (a) the submitted URL is well-formed, (b) custom aliases contain only URL-safe characters and do not conflict with reserved paths, and (c) expiration dates are in the future. Return structured error messages for client display. | C# | High |
| 8 | URL redirection API endpoint | Implement a `GET /{shortCode}` endpoint that looks up the short code, checks expiration and active status, records a click event via the click-tracking service, and returns an HTTP 301/302 redirect to the original URL. Return 404 or 410 for invalid/expired links. | C# | High |
| 9 | Link management API – CRUD operations | Implement REST endpoints for creating (`POST /api/links`), reading (`GET /api/links`, `GET /api/links/{id}`), updating (`PUT /api/links/{id}`), and deleting (`DELETE /api/links/{id}`) short links. Protect mutating operations with appropriate authorization. | C# | High |
| 10 | Analytics API for click data retrieval | Implement `GET /api/links/{id}/analytics` that returns total click count and a time-series breakdown of clicks by date, allowing the frontend to display engagement trends per link. | C# | Medium |
| 11 | Link expiration service / background job | Implement a background service (e.g. `IHostedService`) that periodically marks expired links as inactive so that the redirection endpoint returns the correct status without requiring real-time expiry checks on every request. | C# | Medium |
| 12 | Dashboard layout and navigation | Create the main Blazor layout with a sidebar/top navigation linking to the link list, link creation form, and analytics views. Ensure responsive design suitable for the marketing team's desktop and tablet use. | Blazor | High |
| 13 | Link creation form / component | Build a Blazor form component that accepts a long URL, optional custom alias, and optional expiration date. Integrate with the link management API, display validation errors inline, and show the generated short URL on success. | Blazor | High |
| 14 | Link list view component | Build a Blazor component that fetches and displays all shortened links in a table (short code, original URL, click count, expiration date, status). Support sorting and basic filtering. Provide copy-to-clipboard and delete actions per row. | Blazor | High |
| 15 | Link analytics view component | Build a Blazor component that shows per-link analytics: total clicks and a click-over-time chart. Fetch data from the analytics API and render it using a suitable Blazor-compatible charting library. | Blazor | Medium |
| 16 | Error handling for expired / invalid links | Implement a user-friendly Blazor error page (or component) rendered when the redirection endpoint returns 404 or 410. Display a clear message distinguishing "link not found" from "link expired" scenarios. | Blazor | Medium |
| 17 | Real-time click count updates | Add real-time (or near-real-time) click count refresh to the link list view, using either SignalR or periodic polling, so the marketing team can see engagement without manually refreshing the page. | Blazor | Low |
| 18 | End-to-end redirection flow testing | Write integration/E2E tests covering the full redirection flow: create a link → visit the short URL → verify HTTP redirect → verify click event is recorded in the database. | Both | High |
| 19 | Integration between frontend and backend APIs | Verify that all Blazor components correctly consume the backend REST API: link creation, listing, deletion, and analytics. Add integration tests or Playwright smoke tests to catch contract mismatches early. | Both | Medium |
| 20 | Performance testing for redirection | Benchmark the `GET /{shortCode}` redirection endpoint under realistic load (e.g. using k6 or NBomber). Establish a baseline latency target (e.g. p99 < 100 ms) and document results. | Both | Low |
